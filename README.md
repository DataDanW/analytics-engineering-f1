## Why Formula 1?

Formula 1 data is:
- Relational and messy in its raw form
- Rich enough to model real-world analytics problems
- Familiar, which helps focus on data design rather than domain confusion

It‚Äôs a great sandbox for practising data modelling, SQL structuring, and analytics engineering concepts.

## What this project covers

Across this repository, I explore:

- Working with raw, unmodelled datasets
- Designing fact and dimension tables
- Defining grain and business logic clearly
- Writing clean, maintainable SQL
- Introducing dbt fundamentals for transformation and testing
- Building thin, performant dashboards on top of trusted models

Each step is tied to a specific problem and trade-off.
